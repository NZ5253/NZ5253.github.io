About Me

I'm Naeem Zain Uddin, an Automation & Robotics engineer with a passion for building intelligent systems that bridge software and hardware. My expertise spans reinforcement learning, test automation, embedded systems, and full-stack development.

During my time as a Werkstudent Software Validation at Kandou in Dortmund, I led the development of an Interop Automation framework for Intel/AMD benchmark testing, designing a web-based test execution platform with Python, Flask, SQL, and SSH-based remote execution across distributed lab environments.

At TUV Rheinland, I worked as a Werkstudent in Wireless Research, developing a fully automated Python-based testing framework for Zigbee and Matter CLI devices, replacing manual customer project validation with robust automation pipelines featuring serial control, dynamic command injection, and regex log parsing.

As a Wissenschaftlicher Hilfskraft (HiWi) at the Chair for Reliability Engineering, TU Dortmund, I developed a comprehensive benchmark catalogue of over 85 structured FEM simulation cases, architected MATLAB interfaces, and built Python-based validation tools for data correctness across simulation environments.

My Master's thesis at the Lehrstuhl fur Regelungssystemtechnik focuses on Reinforcement Learning for Autonomous Valet Parking with the Chronos Car, developing a PPO agent with a 14-component reward function, 7-stage training curriculum, and a hybrid NMPC baseline using CasADi/IPOPT achieving sub-centimeter precision, all within ROS 2 and PyTorch.
